# WB Негатив Бот — Автоматическая обработка негативных отзывов на Wildberries

Сервис для селлеров Wildberries, который автоматически отслеживает негативные отзывы и мгновенно отправляет сообщение клиенту через чат покупателей. Управление — через Telegram-бота.

## Что делает сервис

1. **Мониторит новые негативные чаты** — polling WB Buyers Chat API каждые 5 секунд
2. **Верифицирует что это отзыв** — проверяет наличие авто-сообщения WB про отзыв
3. **Отправляет сообщение клиенту** — настраиваемый шаблон с задержкой 3 сек (после авто-сообщения WB)
4. **Мониторит удаление отзывов** — ежечасные снимки, алерт при массовом удалении (5+)
5. **Уведомляет в Telegram** — личка + групповой чат с топиками

## Архитектура

### Мультимагазин

Сервис поддерживает **несколько магазинов (кабинетов WB)** одновременно. Каждый магазин имеет:
- Свой WB API токен
- Своё сообщение для клиентов
- Свой whitelist товаров
- Свой режим (ТЕСТ / БОЕВОЙ)
- Свой cursor для событий
- Свою историю обработки

Магазины добавляются через Telegram-бота (не через .env).

### Polling model

WB Buyers Chat API не поддерживает webhooks. Сервис работает по модели polling:
- Telegram updates — каждую 1 секунду (быстрый отклик на кнопки)
- WB events — каждые 5 секунд (компромисс скорость/лимиты)
- Review snapshots — каждый час

## WB APIs — используемые endpoints

### Buyers Chat API (`buyer-chat-api.wildberries.ru`)

| Метод | Endpoint | Назначение |
|-------|----------|-----------|
| GET   | `/api/v1/seller/events` | Поток событий чатов (polling с cursor) |
| GET   | `/api/v1/seller/chats`  | Список чатов (для верификации + fallback replySign) |
| POST  | `/api/v1/seller/message` | Отправка сообщения (multipart/form-data) |

**Требуемые права токена:** Чат с покупателями

### Content API (`content-api.wildberries.ru`)

| Метод | Endpoint | Назначение |
|-------|----------|-----------|
| POST  | `/content/v2/get/cards/list` | Загрузка каталога товаров (названия для UI) |

**Требуемые права токена:** Контент

### Feedbacks API (`feedbacks-api.wildberries.ru`)

| Метод | Endpoint | Назначение |
|-------|----------|-----------|
| GET   | `/api/v1/feedbacks` | Список отзывов (для мониторинга удалений) |

**Требуемые права токена:** Отзывы и вопросы

> При добавлении магазина нужен токен с правами: **Чат с покупателями + Контент**. Для мониторинга отзывов — дополнительно **Отзывы и вопросы**.

## Бизнес-логика обработки чатов

### Основной pipeline

```
WB создаёт чат (негативный отзыв)
  ↓
Бот обнаруживает isNewChat=true в событиях
  ↓
Атомарная резервация в БД (status=pending) — защита от дублей
  ↓
Проверка whitelist товаров (если настроен)
  ↓
Задержка 3 секунды (ожидание авто-сообщения WB)
  ↓
Верификация: это чат по отзыву? (проверка ключевых слов в lastMessage)
  ↓
  НЕТ → status=skipped, пропуск
  ДА ↓
  ↓
Режим ТЕСТ → уведомление в Telegram, не отправляем
Режим БОЕВОЙ → отправка сообщения в WB + уведомление
  ↓
Сохранение результата в БД (status=sent / error)
```

### Верификация отзывных чатов

WB создаёт чаты не только по отзывам, но и по обычным вопросам клиентов. Бот отправляет сообщение **только** если:
1. В lastMessage чата есть ключевые слова: "отзыв", "оценк", "негативн"
2. Или в событиях чата есть сообщение от seller/system с этими словами

Если WB авто-сообщение не найдено — чат пропускается. Это защита от спама в обычные чаты.

### Cursor и первый запуск

- Cursor — integer timestamp в миллисекундах, уникальный для каждого магазина
- При добавлении нового магазина cursor = текущий момент
- **Старые чаты НЕ обрабатываются** — только будущие события
- После рестарта cursor восстанавливается из БД

### Защита от дублей

Трёхуровневая защита:
1. **Atomic reserve** — `INSERT OR IGNORE` в БД до начала обработки
2. **UNIQUE constraint** — `(store_id, chat_id)` на уровне SQLite
3. **Semaphore** — максимум 3 параллельных обработки на магазин

### Мониторинг отзывов

Раз в час бот:
1. Запрашивает полный список отзывов через Feedbacks API
2. Считает количество по каждому товару (nmID)
3. Сравнивает с предыдущим snapshot
4. Если пропало 5+ отзывов — отправляет алерт в Telegram

## Telegram-бот

### Нижнее меню (Reply Keyboard)

| Кнопка | Действие |
|--------|----------|
| 📊 Статус | Состояние активного магазина: режим, товары, обработано, ошибки |
| ⚙️ Настройки | Inline-меню: режим, сообщение, товары, переименование, удаление |
| 📈 Аналитика | Жалобы по товарам и категориям |
| 🏪 Магазины | Список магазинов, переключение, добавление нового |

### Команды

| Команда | Описание |
|---------|----------|
| `/start` | Главное меню (или онбординг если нет магазинов) |
| `/connect` | Привязать группу/топик для уведомлений (писать в нужном топике) |
| `/disconnect` | Отвязать группу |
| `/stop` | Экстренная остановка всех магазинов |

### Онбординг (добавление магазина)

4 шага:
1. **Название магазина** — произвольное имя для идентификации
2. **API токен** — вставить токен WB (проверяется автоматически)
3. **Сообщение** — выбрать шаблон или написать своё
4. **Товары** — выбрать конкретные или все

### Групповые уведомления

Бот можно добавить в группу Telegram (с топиками):
1. Добавить бота в группу как администратора (право отправки сообщений)
2. Зайти в нужный топик и написать `/connect`
3. Уведомления пойдут в этот топик

В группу приходят:
- Новые обращения клиентов (ещё не отзыв)
- Ответы клиентов на отправленные сообщения
- Алерты об удалении отзывов

Управление (настройки, режимы) — только в личке с ботом.

## Установка

### Локально (Mac/Linux)

```bash
git clone https://github.com/babakov3/wb-chat-bot.git
cd wb-chat-bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Настройка .env

```bash
cp .env.example .env
```

Заполнить:
```env
TELEGRAM_BOT_TOKEN=токен_от_BotFather
TELEGRAM_CHAT_ID=ваш_chat_id
```

> WB токены и настройки магазинов задаются через бота, не через .env.

### Запуск

```bash
python -m app.main
```

Затем написать боту `/start` в Telegram.

## Деплой на сервер (VPS)

### systemd

```ini
[Unit]
Description=WB Chat Bot
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/wb-chat-bot
ExecStart=/opt/wb-chat-bot/venv/bin/python -m app.main
Restart=always
RestartSec=10
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

### Обновление

```bash
cd /opt/wb-chat-bot && git pull && systemctl restart wb-chat-bot
```


## База данных (SQLite)

### Таблицы

| Таблица | Назначение |
|---------|-----------|
| `stores` | Магазины: токен, сообщение, whitelist, режим, группа |
| `store_products` | Каталог товаров магазина (nmID + название) |
| `processed_chats` | Обработанные чаты: статус, клиент, товар, категория |
| `user_state` | Состояние UI пользователя (активный магазин, шаг онбординга) |
| `key_value` | Cursor и другие служебные данные |
| `review_snapshots` | Снимки количества отзывов по товарам (для мониторинга) |
| `notified_events` | Уже отправленные уведомления (дедупликация) |

### Миграции

При запуске storage.py автоматически:
- Создаёт отсутствующие таблицы
- Добавляет новые колонки (ALTER TABLE) если их нет
- Не удаляет существующие данные

## Технические детали

- **Retry**: exponential backoff (2s, 4s, 8s) для 5xx и сетевых ошибок
- **Rate limit**: пауза на Retry-After при 429 от WB
- **Concurrency**: asyncio + semaphore (макс. 3 параллельных обработки на магазин)
- **Отправка сообщений**: multipart/form-data через POST
- **Graceful shutdown**: SIGINT/SIGTERM корректно завершают все задачи
- **Polling intervals**: Telegram 1с, WB events 5с, Reviews 1 час
- **DB**: SQLite с WAL mode и busy_timeout=5000ms

## Категории жалоб

Автоматическая категоризация по ключевым словам в тексте клиента:

| Категория | Ключевые слова |
|-----------|---------------|
| 🔧 Не работает | не работает, сломался, не включается |
| 📦 Брак | брак, дефект, бракованный |
| 🧩 Комплектация | не хватает, некомплект, нет в комплекте |
| 📐 Несоответствие | не соответствует, отличается, другой |
| 🚚 Доставка | доставка, повреждён, упаковка |
| 💰 Возврат | возврат, вернуть деньги, компенсация |
| ❓ Другое | всё остальное |
